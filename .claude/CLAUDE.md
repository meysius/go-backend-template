# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make dev            # hot-reload dev server (requires air)
make build          # compile to bin/app
make test           # run all tests (go test ./...)
go test ./domain/identity/...  # run tests for a single package
make sqlc-gen       # regenerate Go code from SQL queries
make swagger        # regenerate Swagger docs (swag init)
make migrate        # apply pending migrations
make migrate-diff   # generate migration from schema diff (interactive)
make install        # install dev tools: air, sqlc, swag, atlas
```

## Architecture

Gin + PostgreSQL + sqlc + Atlas. Domain-driven slices under `domain/`.

**Layering:** `main.go` → `app/` (wiring) → `controllers/` (HTTP) → `domain/<slice>/` (service + repo) → `db/` (sqlc-generated)

**Domain slices** (`domain/identity/`, `domain/ordering/`): each contains:
- `*_service.go` — business logic, injected into controllers
- `*_repo.go` — repo interface + pgx implementation using sqlc `db.Queries`
- `*_schema.sql` — canonical table DDL (source of truth for Atlas migrations)
- `*_queries.sql` — SQL queries consumed by sqlc

**`db/` is fully generated by sqlc — never edit files in `db/` manually.** Run `make sqlc-gen` after changing `*_queries.sql` or `*_schema.sql`.

**`app/app.go`** wires all dependencies (pool, repos, services, controllers) and mounts routes. New domain slices follow the same pattern: add repo+service in `domain/`, controller in `controllers/`, wire in `app/app.go`.

**Schema changes:** edit `domain/<slice>/<slice>_schema.sql` → `make migrate-diff` → review `migrations/` → `make migrate` → `make sqlc-gen`.

## Key Conventions

- Config via `.env` file (loaded by godotenv); required vars: `DATABASE_NAME`, `DATABASE_USER`, `DATABASE_PASS`
- Structured logging with `log/slog`; use `helpers.LoggerFrom(ctx)` for request-scoped logger (carries request_id)
- Swagger annotations on controller handler funcs; regenerate with `make swagger`
- Controllers define their own request/response types and map to/from `db.*` types
